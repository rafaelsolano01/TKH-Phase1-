1. REFLECTED XSS
* Payload Used: <script>alert('XSS Reflected!')</script>
* Concept: Why does this attack disappear when you refresh the page?
Answer: This is a non-persistent attack. The payload is only executed when it is processed as a URL parameter (q=). Once the page is refreshed without that parameter, the server no longer "reflects" the malicious script back to the browser.

2. STORED XSS (Cookie Theft)
* Cookie Value Stolen: session_id=SECRET_ADMIN_TOKEN_ABC123
* Payload Used: <script>alert("Cookie Stealer: " + document.cookie)</script>

3. CROSS-SITE REQUEST FORGERY (CSRF)
* Malicious URL Crafted: http://127.0.0.1:8082/transfer?to=Attacker&amount=5000
* Concept: What is the primary defense against CSRF?
Answer: The primary defense is an Anti-CSRF token—a unique, unpredictable value generated by the server and validated for every state-changing request. Secondary defenses include using the SameSite cookie attribute and switching from GET to POST requests for sensitive actions.
